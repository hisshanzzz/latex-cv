B — Asgardeo login (WSO2)

English Pro Hub is on the EU region. Use EU links, not the US ones: Console https://console.eu.asgardeo.io/t/englishprohub · My Account https://myaccount.eu.asgardeo.io/t/englishprohub · API baseUrl https://api.eu.asgardeo.io/t/englishprohub. The Organization UUID is not the org name in /t/....

  1. Open the EU console: console.eu.asgardeo.io/t/englishprohub.
  2. Org name from the URL is englishprohub (already set in js/config.js with region: "eu").
  3. Applications → New → Single-Page Application named SpellQuest.
  4. Under Protocol, set these exactly:
    • Authorized redirect URLs (one per line):
      • https://spellquest-englishprohub.netlify.app/
      • http://127.0.0.1:8888/
      • http://localhost:8888/
    • Allowed origins:
      • https://spellquest-englishprohub.netlify.app
      • http://127.0.0.1:8888
      • http://localhost:8888
  5. Copy Client ID from the application (already set: 7CLd15CM1Xw0HB6VKA6ZtFJaBX8a).
  6. In js/config.js confirm:
    • asgardeo.org = englishprohub
    • asgardeo.region = eu
    • asgardeo.baseUrl = https://api.eu.asgardeo.io/t/englishprohub
    • asgardeo.clientId = Client ID
  7. In Netlify → Site settings → Environment variables, set:
    • ASGARDEO_ORG = englishprohub
    • ASGARDEO_REGION = eu
    • TEACHER_PIN = already set (246810)
  8. Redeploy after any config change.

C — Hosting + APIs (Netlify)

We use Netlify Functions + Blobs for progress sync, leaderboard, and teacher tools.

  1. cd spellquest
  2. npm install
  3. npx netlify login
  4. Local API test: npx netlify dev (usually http://127.0.0.1:8888)
  5. Production: npx netlify deploy --prod --dir=.

What each API does

  • /api/progress — save/load progress when signed in
  • /api/leaderboard — public scores
  • /api/teacher — classroom codes + board overview (PIN protected)

Choreo later?

If you outgrow Netlify Functions, the same API routes can be re-hosted on WSO2 Choreo. Start here so students get a working link first.
